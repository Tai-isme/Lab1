<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Exception → Response | Unhandled exception details cross from internal code to HTTP response — must sanitize in production |
| Middleware → Pipeline | Exception middleware sits early in pipeline — misplacement could cause double-wrapping or missed exceptions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | ExceptionHandlingMiddleware | mitigate | D-02: Production mode returns only generic "An internal server error occurred" — no exception type, message, or stack trace. `IsDevelopment()` gate prevents info leak. |
| T-08-02 | Denial of Service | ExceptionHandlingMiddleware | mitigate | `context.Response.HasStarted` guard prevents writing to already-started responses, avoiding runtime exceptions in the middleware itself. |
| T-08-03 | Tampering | ResponseEnvelopeFilter + Middleware | accept | Exception middleware returns `ApiResponse<object>` directly (bypassing filter via ObjectResult). Filter's existing `IsGenericType` check for `ApiResponse<>` prevents double-wrapping. Low risk — no user input reaches this path. |
</threat_model>
